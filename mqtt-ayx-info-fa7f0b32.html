MQTT 主题与数据
在 ayx 视角下的表达

从边缘采集到云端分发，ayx 提供一种聚焦于主题结构与信道效率的技术映射。不追求大而全，而是将连接语义收敛到核心字段与路由规则。

主题（Topic）是 MQTT 协议中定义消息类别的标识符，它决定了消息如何从发布者路由到订阅者。在 ayx 的设计语境里，主题不再只是斜杠分隔的字符串，而是一种带有层级语义的轻量命名空间。通过严格限定通配符使用、控制 topic 长度，可有效降低 broker 匹配开销。

基于 ayx 的实践，我们将主题拆解为 域 · 组 · 设备 · 信号 四个基础层次。这种固定结构可以避免主题爆炸，同时保留足够的表达弹性。例如在工业传感器网络中，每个节点都能在预设的映射规则下快速完成发布与订阅，不需要额外的主题映射表。

场域 · 边缘

设备状态聚合

通过 ayx 主题模板，将温度、振动、功率等信号编排到统一前缀下，订阅端只需监听 +/status/# 即可获得所有节点的最新快照，大幅减少冗余连接。

场域 · 指令

双向命令下发

利用 MQTT 的保留消息与 QoS 1 语义，在 ayx 主题空间中设定 cmd/ 分支。执行器重启后自动同步最近一条指令，不依赖中心数据库。

场域 · 桥接

跨协议翻译

当 MQTT 需要与 HTTP / CoAP 系统交互时，ayx 提供轻量主题转换层，将外部请求映射为内部主题，保持端到端语义一致性。

关于 ayx 主题设计的几点说明

· 主题数量控制： 建议单个 broker 下的活跃主题不超过 2000 个，ayx 推荐按功能域拆分，避免全局通配符滥用。

· 命名约束： 主题中禁止使用空格、中文及特殊符号（+ # $ 仅用于通配符定义），层级推荐用 a-z 与连字符 -。

· 安全考虑： 通过 ayx 预设的 ACL 模板，可以按主题前缀限制客户端的发布/订阅权限，而非依赖 IP 白名单。

更完整的参数说明与示例可参考 ayx 项目文档。
